## 🔍 トラブルシューティングの経緯

### 問題
GTMプレビューモードで「デバッグ可能なタグがない」というエラーが発生

### 試した方法

1. **オプション1: `next/script`コンポーネント** → 失敗
   - Next.jsの静的エクスポートでは特殊な形式でレンダリングされる
   - GTMプレビューモードが認識できなかった

2. **オプション2: ビルド後のHTML注入** → 成功
   - HTMLファイルに直接GTMコードを注入
   - GTM自体は正常に動作

3. **ブラウザ拡張機能の影響を発見**
   - 通常モード: 失敗（広告ブロッカーが原因）
   - シークレットモード: 成功

### 根本原因
**ブラウザ拡張機能（広告ブロッカー）がGTMプレビューモードをブロックしていた**

## 🎯 重要な知見

### 1. GTMプレビューモードのテスト方法

**通常モードでは失敗する原因：**
- 広告ブロッカー（uBlock Origin、AdBlock等）
- プライバシー保護拡張機能
- これらがGTMデバッグスクリプトをブロックする

**推奨テスト方法：**
- ✅ **シークレットモードを使用**
- ✅ または、すべての拡張機能を一時的に無効化

### 2. 実際のユーザーへの影響

**重要**: 広告ブロッカーがあってもGTM自体は動作します

- ユーザーが広告ブロッカーを使用していても、GTMは正常に動作
- データ収集は正常に行われる
- 問題があるのは**開発者のプレビューモード接続**のみ

## 🚀 今後の使用方法

### GTMでタグを設定する

1. Google Tag Manager (https://tagmanager.google.com/) にログイン
2. コンテナ「GTM-KKP36CQQ」を選択
3. タグを追加（GA4、広告タグ等）
4. 変更を「公開」

### プレビューモードでテスト

1. **シークレットモードでブラウザを開く**
2. Google Tag Managerで「プレビュー」をクリック
3. URLを入力: `https://jussssss37.github.io/kumakichi/`
4. 「Connect」をクリック
5. デバッグパネルでタグの動作を確認

### デプロイ

コードの変更は不要です。以下の場合のみビルド＆デプロイ：
- サイトのコンテンツを更新した場合
- GTMコンテナIDを変更する場合

```bash

npm run build

# デプロイ
git add .
git commit -m "Update content"
```

## 📞 トラブル時の対応

問題が発生した場合：

1. **GTMが動作しているか確認**
   ```javascript
   // ブラウザのConsoleで実行
   window.dataLayer
   ```
   配列が返ってくれば正常

2. **プレビューモードで接続できない**
   → シークレットモードを使用

3. **データが収集されない**
   → GTMコンソールでタグの設定を確認

4. **デプロイ後にGTMが消えた**
   → `npm run build` を再実行（inject-gtm.jsが実行される）
